# Auto-Insight Platform - 핵심 다이어그램

면접용 핵심 다이어그램 3개 (한국어 + 세로형)

## 사용 방법

1. **Mermaid Live Editor** 접속: https://mermaid.live
2. `.mmd` 파일 열기 → 전체 복사 → 붙여넣기
3. PNG/SVG 다운로드

---

## 1️⃣ 시스템 아키텍처 (System Architecture)

**파일**: `system_architecture.mmd`

**30초 설명**:
```
"사용자가 파일을 업로드하면
데이터를 처리하고 분석한 후
차트와 리포트를 생성합니다.
GPT로 인사이트를 추출하고
SQLite에 저장합니다."
```

**구조**:
```
사용자
  ↓
데이터 입력 (업로드 + 크롤링)
  ↓
데이터 처리 (로딩 + 정제 + 검증)
  ↓
분석 엔진 (RFM + 감성 + 매출)
  ↓ ↘
GPT   결과 출력
       ↓
     사용자
```

**노드 수**: 7개 (극단적 단순화)

---

## 2️⃣ 데이터 파이프라인 / ETL 흐름

**파일**: `data_pipeline_etl.mmd`

**30초 설명**:
```
"ETL 파이프라인입니다.
데이터를 수집하고(Extract),
정제 및 변환한 다음(Transform),
분석 후 리포트로 내보냅니다(Load)."
```

**흐름** (세로 방향):
```
📥 데이터 수집 (EXTRACT)
      ↓
📂 데이터 로딩
      ↓
✓ 데이터 검증
      ↓
🧹 데이터 정제
      ↓
⚙️ 데이터 변환
      ↓
🔀 데이터 타입 감지
      ↓
📊 데이터 분석 (ANALYZE)
      ↓
📈 시각화
      ↓
📤 결과 내보내기 (LOAD)
```

**노드 수**: 9개 (완전 직선 흐름)

---

## 3️⃣ 데이터베이스 스키마 (ERD)

**파일**: `database_schema_erd.mmd`

**30초 설명**:
```
"3개 테이블을 설계했습니다.
REVIEWS: 크롤링한 리뷰 데이터
TRANSACTIONS: 거래 데이터
CUSTOMERS: RFM 분석 결과
거래 데이터를 분석해서 고객 세그먼트를 생성합니다."
```

**테이블 구조**:

| 테이블 | 설명 | 주요 필드 |
|--------|------|-----------|
| **REVIEWS** | 리뷰 원본 데이터 | 출처, 리뷰내용, 평점, 작성일, 감성분석결과 |
| **TRANSACTIONS** | 거래 원본 데이터 | 고객ID, 거래일, 상품명, 수량, 단가 |
| **CUSTOMERS** | RFM 분석 결과 | 고객ID, 최근구매일, 구매빈도, 구매금액, 고객세그먼트 |

**관계**: TRANSACTIONS → CUSTOMERS (1:N, RFM분석)

## 📊 면접 시나리오

### ⏱️ 3분 (빠른 설명)
**사용**: 시스템 아키텍처 1개만

"파일 업로드 → 자동 분석 → 리포트 생성 플랫폼입니다.
GPT로 인사이트를 추출하고 SQLite에 저장합니다."

---

### ⏱️ 5분 (표준 설명)
**사용**: 시스템 아키텍처 + 데이터 파이프라인

"(시스템 아키텍처 보여주며) 전체 구조는 이렇고,
(데이터 파이프라인 보여주며) 데이터는 ETL로 처리됩니다.
자동으로 데이터 타입을 감지해서 적절한 분석을 수행합니다."

---

### ⏱️ 10분 (심층 설명)
**사용**: 전체 3개

"(시스템 아키텍처) 시스템 구조입니다.
(데이터 파이프라인) 데이터 흐름은 ETL 패턴을 따릅니다.
(ERD) SQLite로 3개 테이블을 설계했고,
거래 데이터를 분석해서 고객 세그먼트를 자동 생성합니다."

1. **시스템 아키텍처** (1-2분)
   - "전체 구조부터 설명드리겠습니다"
