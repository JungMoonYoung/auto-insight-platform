## 1. 프로젝트 개요

### 1.1 프로젝트 목표

비개발자도 쉽게 사용할 수 있는 **제로 코딩 데이터 분석 플랫폼** 구축

**핵심 가치**:
- 파일 업로드만으로 전문가 수준의 분석 제공
- AI 기반 자동 인사이트 생성
- 인터랙티브 대시보드 + 다운로드 가능한 리포트

### 1.2 타겟 사용자

- 마케팅/기획 실무자 (비개발자)
- 소상공인 및 중소기업 운영자
- 데이터 분석가 (빠른 프로토타이핑)

### 1.3 주요 기능

| 분석 타입 | 주요 기능 | 비즈니스 가치 |
|----------|----------|--------------|
| **E-commerce** | RFM 분석, 고객 세분화 | VIP 고객 식별, 이탈 방지 전략 |
| **리뷰 분석** | 감성 분석, 토픽 모델링 | 제품 개선점 발견, VOC 분석 |
| **판매 분석** | 시계열 트렌드, 상품 순위 | 매출 패턴 파악, 재고 최적화 |

### 3.3 데이터 흐름

```
1. 데이터 입력
   ├─ 파일 업로드 (CSV/Excel)
   └─ 크롤링 (로컬) or 샘플 선택 (배포)

2. 자동 매핑
   ├─ 컬럼명 유사도 매칭 (FuzzyWuzzy)
   ├─ 샘플 데이터 통계 분석
   └─ 신뢰도 < 80% → 수동 매핑 UI

3. 전처리
   ├─ 결측치 처리 (auto 전략)
   ├─ 이상치 처리 (IQR)
   └─ 날짜 파생 변수 생성

4. 분석 (타입별 자동 분기)
   ├─ E-commerce → RFM + K-Means
   ├─ Review → 키워드 감성 + GPT (부정 100개)
   └─ Sales → 시계열 + 상품 순위

5. 시각화
   ├─ Plotly 인터랙티브 차트
   └─ 인사이트 자동 생성

6. 출력
   ├─ CSV (원본 + 분석 결과)
   └─ HTML 리포트
```

## 부록: 용어 정의

| 용어 | 정의 |
|------|------|
| RFM | Recency, Frequency, Monetary (고객 세분화 기법) |
| K-Means | 비지도 학습 군집화 알고리즘 |
| Silhouette Score | 군집화 품질 평가 지표 (0~1, 높을수록 좋음) |
| TF-IDF | Term Frequency-Inverse Document Frequency (키워드 중요도) |
| LDA | Latent Dirichlet Allocation (토픽 모델링) |
| GPT-4o-mini | OpenAI 경량 모델 (비용 저렴, 빠른 응답) |
| Streamlit Cloud | Streamlit 무료 호스팅 플랫폼 |
| session_state | Streamlit 세션 간 데이터 공유 메커니즘 |
| 하이브리드 크롤링 | 로컬=실제 크롤링, 배포=샘플 데이터 |
